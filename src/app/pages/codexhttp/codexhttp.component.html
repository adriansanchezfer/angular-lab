Codex HTTP

Una peticion HTTP es la conversacion entre tu frontend y un servidor: Angular envia un metodo, una URL, cabeceras y a veces un cuerpo; la API responde con un codigo de estado, cabeceras y datos. La parte importante en frontend no es solo "hacer fetch": tambien es representar carga, errores, reintentos, datos vacios y transformaciones. Aqui usamos DummyJSON, una API gratuita de prototipado con productos, usuarios, posts y todos.
